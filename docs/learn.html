Chapter 01 · 기본 자료구조Basic DS
Array vs ContiguousArray
CoW 메커니즘과 메모리 레이아웃 — Swift 배열의 진짜 동작
CoW mechanism and memory layout — how Swift arrays really work
🗂
DataStructures.playground
자료구조 Ch01–10 · Learn + Practice 20페이지 · Xcode에서 바로 실행DS Ch01–10 · 20 pages Learn + Practice · Run in Xcode
⬇ KO ⬇ EN
개념CONCEPT

Array는 Swift 표준 배열로 Copy-on-Write(CoW)를 사용합니다. 복사가 일어나도 실제 메모리 복사는 쓰기 시점에만 발생해 불필요한 복사를 방지합니다. 요소가 클래스 타입이면 NSArray 브리징이 가능하지만 그만큼 오버헤드가 있습니다.

ContiguousArray는 항상 연속 메모리를 보장합니다. 클래스/프로토콜 타입 요소를 다룰 때 Array보다 빠르며 브리징 오버헤드가 없습니다. 구조체(Int, Double 등) 원소에서는 Array와 성능이 동일합니다.

Array is Swift's standard array type using Copy-on-Write (CoW). Even after copying, the actual memory copy only happens on the first write, avoiding unnecessary duplication. If elements are class types, NSArray bridging is possible but adds overhead.

ContiguousArray always guarantees contiguous memory. It's faster than Array for class/protocol element types and has no bridging overhead. For struct types (Int, Double, etc.) performance is identical to Array.

Swift CODE
// CoW 동작 확인
var a = [1, 2, 3]
var b = a              // 복사 X — 같은 버퍼 공유
b[0] = 99             // 이 시점에 실제 복사 발생
print(a[0])           // 1 — a 영향 없음

// ContiguousArray: 클래스 원소에서 빠름
var ca: ContiguousArray<SomeClass> = []
ca.reserveCapacity(100)  // 재할당 최소화

// 성능 중요할 때: withUnsafeBufferPointer
a.withUnsafeBufferPointer { ptr in
    // C 함수에 직접 포인터 전달 — 복사 없음
}

Chapter 02 · 기본 자료구조Basic DS
Linked List
노드를 포인터로 연결 — O(1) 삽입/삭제의 비밀
Connecting nodes with pointers — the secret of O(1) insert/delete
개념CONCEPT

Linked List는 Node 객체들이 next 포인터로 연결된 선형 자료구조입니다. Array와 달리 메모리가 연속하지 않아도 되므로 노드 위치를 알고 있을 때 삽입/삭제가 O(1)입니다.

단방향(Singly): next 포인터만 보유. tail 삭제 O(n). 양방향(Doubly): prev/next 모두 보유. tail 삭제 O(1)이나 메모리 2배, retain cycle 주의.

A Linked List is a linear data structure where Node objects are connected via next pointers. Unlike an Array, memory doesn't need to be contiguous, so insertion/deletion is O(1) when the node reference is known.

Singly: only next pointer. Tail deletion is O(n). Doubly: has both prev and next. Tail deletion is O(1), but uses 2x memory — beware of retain cycles.

Swift CODE
class Node<T> {
    var value: T
    var next: Node<T>?
    init(_ value: T) { self.value = value }
}

class LinkedList<T> {
    var head: Node<T>?
    var tail: Node<T>?

    func append(_ value: T) {
        let node = Node(value)
        if let tail { tail!.next = node } else { head = node }
        tail = node
    }

    func removeFirst() -> T? {
        defer { head = head?.next; if head == nil { tail = nil } }
        return head?.value
    }
}

Chapter 03 · 기본 자료구조Basic DS
Stack / Queue
LIFO와 FIFO — 알고리즘의 기본 빌딩 블록
LIFO and FIFO — the fundamental building blocks of algorithms
개념CONCEPT

Stack(LIFO): push로 쌓고 pop으로 꺼냅니다. 함수 호출 스택, Undo, 괄호 검증에 사용됩니다.

Queue(FIFO): enqueue로 넣고 dequeue로 꺼냅니다. BFS, 작업 스케줄링, iOS 메시지 큐에 사용됩니다. Array 기반 Queue는 dequeue가 O(n)이므로 두 Stack으로 O(1) amortized Queue를 만들 수 있습니다.

Stack (LIFO): push to add, pop to remove. Used for function call stacks, Undo, and bracket validation.

Queue (FIFO): enqueue to add, dequeue to remove. Used for BFS, task scheduling, and iOS message queues. Array-based Queue has O(n) dequeue, so use two Stacks for O(1) amortized Queue.

Swift CODE
// Stack
struct Stack<T> {
    private var storage: [T] = []
    mutating func push(_ v: T) { storage.append(v) }
    mutating func pop() -> T? { storage.popLast() }
    var top: T? { storage.last }
}

// 두 Stack으로 O(1) amortized Queue
struct Queue<T> {
    private var inbox: [T] = [], outbox: [T] = []
    mutating func enqueue(_ v: T) { inbox.append(v) }
    mutating func dequeue() -> T? {
        if outbox.isEmpty { outbox = inbox.reversed(); inbox = [] }
        return outbox.popLast()
    }
}

Chapter 04 · 기본 자료구조Basic DS
Deque
링 버퍼로 양쪽 모두 O(1) — Double-Ended Queue
Ring buffer for O(1) on both ends — Double-Ended Queue
개념CONCEPT

Deque(덱)은 앞뒤 양쪽에서 O(1) 삽입/삭제가 가능한 자료구조입니다. Stack과 Queue를 모두 대체할 수 있습니다.

링 버퍼(Circular Buffer)로 구현 시 고정 크기 배열에서 head/tail 인덱스만 이동합니다. Swift의 Collections 패키지 Deque가 이 방식이며 가장 효율적입니다.

A Deque (Double-Ended Queue) allows O(1) insertion/deletion at both front and back. It can replace both Stack and Queue.

Implemented as a Ring Buffer: only head/tail indices move in a fixed-size array. Swift's Collections package Deque uses this approach and is the most efficient.

Swift CODE
// 링 버퍼 기반 Deque (단순화 버전)
struct Deque<T> {
    private var buf: [T?]
    private var head = 0, count = 0
    private var cap: Int { buf.count }

    init(capacity: Int = 8) { buf = Array(repeating: nil, count: capacity) }

    mutating func pushFront(_ v: T) {
        head = (head - 1 + cap) % cap
        buf[head] = v; count += 1
    }
    mutating func pushBack(_ v: T) {
        buf[(head + count) % cap] = v; count += 1
    }
    mutating func popFront() -> T? {
        guard count > 0 else { return nil }
        defer { buf[head] = nil; head = (head + 1) % cap; count -= 1 }
        return buf[head]
    }
}

Chapter 05 · 기본 자료구조Basic DS
Binary Tree
전위 · 중위 · 후위 · 레벨 순회 — 트리의 모든 것
Pre · In · Post · Level-order traversal — everything about trees
개념CONCEPT

각 노드가 최대 2개의 자식(left, right)을 갖는 계층적 자료구조. 재귀 구조라 재귀 알고리즘과 찰떡입니다.

전위(Pre-order) 루트→좌→우 | 중위(In-order) 좌→루트→우 | 후위(Post-order) 좌→우→루트 | BFS 층별 탐색

A hierarchical structure where each node has at most 2 children (left, right). The recursive structure pairs perfectly with recursive algorithms.

Pre-order root→left→right | In-order left→root→right | Post-order left→right→root | BFS level-by-level

Swift CODE
class TreeNode<T> {
    var val: T; var left, right: TreeNode<T>?
    init(_ v: T) { val = v }
}

func inorder<T>(_ node: TreeNode<T>?) -> [T] {
    guard let node else { return [] }
    return inorder(node.left) + [node.val] + inorder(node.right)
}

// BFS — 레벨 순회
func bfs<T>(_ root: TreeNode<T>?) -> [[T]] {
    guard let root else { return [] }
    var q = [root], result: [[T]] = []
    while !q.isEmpty {
        result.append(q.map { $0.val })
        q = q.flatMap { [$0.left, $0.right].compactMap { $0 } }
    }
    return result
}

Chapter 06 · 기본 자료구조Basic DS
Binary Search Tree
삽입 · 탐색 · 삭제(3케이스) — 정렬된 트리의 힘
Insert · Search · Delete (3 cases) — the power of sorted trees
개념CONCEPT

모든 노드가 왼쪽 < 현재 < 오른쪽 규칙을 만족하는 이진 트리. 탐색이 절반씩 줄어 평균 O(log n)입니다.

삭제 3케이스: ① 자식 없음 → 그냥 삭제 ② 자식 1개 → 자식으로 대체 ③ 자식 2개 → 오른쪽 서브트리 최솟값(Successor)으로 대체.

A binary tree where every node satisfies left child < current < right child. This property halves the search space, giving average O(log n).

Delete 3 cases: ① No children → just delete ② One child → replace with child ③ Two children → replace with in-order successor (min of right subtree).

Swift CODE
func insert(_ node: TreeNode<Int>?, _ val: Int) -> TreeNode<Int> {
    guard let node else { return TreeNode(val) }
    if val < node.val { node.left  = insert(node.left,  val) }
    else             { node.right = insert(node.right, val) }
    return node
}

func minNode(_ n: TreeNode<Int>) -> TreeNode<Int> {
    var cur = n; while let l = cur.left { cur = l }; return cur
}

func delete(_ node: TreeNode<Int>?, _ val: Int) -> TreeNode<Int>? {
    guard let node else { return nil }
    if      val < node.val { node.left  = delete(node.left,  val) }
    else if val > node.val { node.right = delete(node.right, val) }
    else {
        if node.left  == nil { return node.right }
        if node.right == nil { return node.left  }
        let successor = minNode(node.right!)
        node.val   = successor.val
        node.right = delete(node.right, successor.val)
    }
    return node
}

Heap / Priority Queue
siftUp · siftDown — 최대/최솟값을 항상 O(1)에
siftUp · siftDown — always O(1) access to max/min
개념CONCEPT

완전 이진 트리를 배열에 저장. 인덱스 i의 부모: (i-1)/2, 좌자식: 2i+1, 우자식: 2i+2.

Max Heap: 부모 ≥ 자식. Min Heap: 부모 ≤ 자식. Priority Queue = Heap. iOS OperationQueue.qualityOfService, Dijkstra 핵심.

A complete binary tree stored as an array. Index i: parent at (i-1)/2, left child at 2i+1, right child at 2i+2.

Max Heap: parent ≥ children. Min Heap: parent ≤ children. Priority Queue = Heap. Used in iOS OperationQueue.qualityOfService and Dijkstra's algorithm.

Swift CODE
struct MinHeap {
    var data: [Int] = []

    mutating func push(_ v: Int) {
        data.append(v); siftUp(data.count - 1)
    }
    mutating func pop() -> Int? {
        guard !data.isEmpty else { return nil }
        data.swapAt(0, data.count - 1)
        let v = data.removeLast()
        siftDown(0); return v
    }
    private mutating func siftUp(_ i: Int) {
        var i = i
        while i > 0 {
            let p = (i - 1) / 2
            if data[i] < data[p] { data.swapAt(i, p); i = p } else { break }
        }
    }
    private mutating func siftDown(_ i: Int) {
        var i = i
        while 2*i+1 < data.count {
            var s = 2*i+1
            if s+1 < data.count && data[s+1] < data[s] { s += 1 }
            if data[s] < data[i] { data.swapAt(i, s); i = s } else { break }
        }
    }
}

인접행렬 vs 인접리스트 · BFS · DFS — 관계를 모델링

개념CONCEPT

정점(Vertex)과 간선(Edge)으로 구성. 방향/무방향, 가중치 유무로 다양한 종류가 있습니다.

인접행렬: O(1) 간선 확인, O(V²) 공간 — 밀집 그래프. 인접리스트: O(V+E) 공간 — 희소 그래프. BFS: 최단 경로(가중치 없음). DFS: 사이클 감지, 위상정렬.

Composed of vertices and edges. Comes in directed/undirected and weighted/unweighted varieties.

Adjacency matrix: O(1) edge check, O(V²) space — for dense graphs. Adjacency list: O(V+E) space — for sparse graphs. BFS: shortest path (unweighted). DFS: cycle detection, topological sort.

Swift CODE
// 인접 리스트 그래프
var graph = [[Int]](repeating: [], count: n)
graph[0].append(1)  // 0 → 1

// BFS
func bfs(start: Int) -> [Int] {
    var visited = Set<Int>(), queue = [start], order: [Int] = []
    visited.insert(start)
    while !queue.isEmpty {
        let v = queue.removeFirst(); order.append(v)
        for next in graph[v] where !visited.contains(next) {
            visited.insert(next); queue.append(next)
        }
    }
    return order
}

// DFS (재귀)
func dfs(_ v: Int, _ visited: inout Set<Int>) {
    visited.insert(v)
    for next in graph[v] where !visited.contains(next) { dfs(next, &visited) }
}

func twoSum(_ nums: [Int], _ target: Int) -> [Int] {
    var seen: [Int: Int] = [:]  // value → index
    for (i, n) in nums.enumerated() {

        seen[n] = i
    }
    return []
}

자동완성 DFS · O(m) 탐색 — 문자열 특화 트리

개념CONCEPT

문자열을 글자 단위로 트리에 저장. 검색 길이 m이면 O(m)으로 탐색. 해시 충돌 없이 정확합니다.

각 노드: [Character: TrieNode] 자식 딕셔너리 + isEnd 플래그. 자동완성은 prefix까지 내려간 뒤 DFS로 모든 단어 수집.

Stores strings character by character in a tree. Search length m → O(m) lookup. No hash collisions — always exact.

Swift CODE
class TrieNode {
    var children: [Character: TrieNode] = [:]
    var isEnd = false
}

class Trie {
    let root = TrieNode()

    func insert(_ word: String) {
        var cur = root
        for ch in word {
            if cur.children[ch] == nil { cur.children[ch] = TrieNode() }
            cur = cur.children[ch]!
        }
        cur.isEnd = true
    }

    func autocomplete(_ prefix: String) -> [String] {
        var cur = root
        for ch in prefix { guard let next = cur.children[ch] else { return [] }; cur = next }
        var results: [String] = []
        dfs(cur, prefix, &results)
        return results
    }

    private func dfs(_ node: TrieNode, _ current: String, _ out: inout [String]) {
        if node.isEnd { out.append(current) }
        for (ch, child) in node.children { dfs(child, current + String(ch), &out) }
    }
}

AVL Tree
높이 균형 보장 — 4가지 회전으로 O(log n) 유지
Height balance guaranteed — O(log n) via 4 rotations
🗂
DataStructures.playground
자료구조 Ch01–10 포함 · 심화 챕터는 웹 페이지에서 학습하세요Includes DS Ch01–10 · Study advanced chapters on this page
⬇ KO ⬇ EN
개념CONCEPT

모든 노드에서 좌/우 서브트리 높이 차이(Balance Factor)가 최대 1인 자기 균형 BST. BF = height(left) - height(right), BF가 ±2가 되면 회전으로 복원합니다.

4가지 회전: LL(우회전) | RR(좌회전) | LR(좌회전→우회전) | RL(우회전→좌회전)

A self-balancing BST where the height difference (Balance Factor) between left and right subtrees is at most 1 at every node. BF = height(left) - height(right). When BF reaches ±2, rotations restore balance.

4 rotation cases: LL (right rotation) | RR (left rotation) | LR (left then right) | RL (right then left)

Swift CODE
class AVLNode {
    var val: Int, height = 1
    var left, right: AVLNode?
    init(_ v: Int) { val = v }
}

func h(_ n: AVLNode?) -> Int { n?.height ?? 0 }
func bf(_ n: AVLNode) -> Int { h(n.left) - h(n.right) }
func update(_ n: AVLNode) { n.height = 1 + max(h(n.left), h(n.right)) }

// 우회전 (LL 케이스)
func rotateRight(_ y: AVLNode) -> AVLNode {
    let x = y.left!
    y.left = x.right; x.right = y
    update(y); update(x); return x
}

func balance(_ n: AVLNode) -> AVLNode {
    update(n)
    if bf(n) > 1 {  // 왼쪽 무거움
        if bf(n.left!) < 0 { n.left = rotateLeft(n.left!) }  // LR
        return rotateRight(n)
    }
    if bf(n) < -1 {  // 오른쪽 무거움
        if bf(n.right!) > 0 { n.right = rotateRight(n.right!) }  // RL
        return rotateLeft(n)
    }
    return n
}

5가지 규칙으로 느슨한 균형 — Swift OrderedSet 내부

개념CONCEPT

각 노드가 Red/Black 색상을 가지며 5가지 규칙으로 높이를 O(log n)으로 보장하는 균형 BST.

Segment Tree
구간 쿼리 O(log n) — 합계 · 최솟값 · 최댓값 즉시 조회
Range queries in O(log n) — instant sum · min · max
개념CONCEPT

배열의 구간(range) 쿼리를 빠르게 처리. 배열 크기 n에 대해 2n 크기 배열로 표현 (Heap과 유사한 인덱스 구조). 리프가 원소, 내부 노드가 구간 집계값.

Handles range queries efficiently. Using an array of size 2n for an input of size n (similar index structure to Heap). Leaves store elements, internal nodes store aggregated range values.

Swift CODE
struct SegTree {
    let n: Int
    var tree: [Int]

    init(_ arr: [Int]) {
        n = arr.count; tree = Array(repeating: 0, count: 4*n)
        build(arr, 1, 0, n-1)
    }

    mutating func build(_ arr: [Int], _ node: Int, _ l: Int, _ r: Int) {
        if l == r { tree[node] = arr[l]; return }
        let m = (l+r)/2
        build(arr, 2*node, l, m); build(arr, 2*node+1, m+1, r)
        tree[node] = tree[2*node] + tree[2*node+1]
    }

    func query(_ node: Int, _ l: Int, _ r: Int, _ ql: Int, _ qr: Int) -> Int {
        if qr < l || r < ql { return 0 }
        if ql <= l && r <= qr { return tree[node] }
        let m = (l+r)/2
        return query(2*node, l, m, ql, qr) + query(2*node+1, m+1, r, ql, qr)
    }
}

Union-Find
연결 여부를 거의 O(1)에 — Kruskal MST의 핵심
Near-O(1) connectivity check — the core of Kruskal MST
개념CONCEPT

find(x): x가 속한 집합의 대표(root) 반환. union(x,y): x와 y의 집합을 합침.

Path Compression: find 시 경유 노드를 root에 직접 연결. Union by Rank: 낮은 트리를 높은 트리 아래 붙임. 두 최적화 합치면 O(α(n)) ≈ O(1).

find(x): returns the representative (root) of the set x belongs to. union(x,y): merges the sets of x and y.

Path Compression: during find, connect traversed nodes directly to root. Union by Rank: attach shorter tree under taller tree. Together: O(α(n)) ≈ O(1).

Swift CODE
struct UnionFind {
    var parent, rank: [Int]

    init(_ n: Int) {
        parent = Array(0..<n); rank = Array(repeating: 0, count: n)
    }

    mutating func find(_ x: Int) -> Int {
        if parent[x] != x { parent[x] = find(parent[x]) }  // Path Compression
        return parent[x]
    }

    mutating func union(_ x: Int, _ y: Int) {
        let rx = find(x), ry = find(y)
        if rx == ry { return }
        if rank[rx] < rank[ry] { parent[rx] = ry }        // Union by Rank
        else if rank[rx] > rank[ry] { parent[ry] = rx }
        else { parent[ry] = rx; rank[rx] += 1 }
    }
}

개념CONCEPT

Quick Sort: pivot 기준 분할 정복. 평균 O(n log n), 최악 O(n²). 캐시 친화적이라 실제로 가장 빠름. Swift sort()는 IntroSort(Quick+Heap+Insertion 혼합).

Merge Sort: 항상 O(n log n), 안정 정렬(Stable). 추가 O(n) 메모리. 연결 리스트 정렬 최적.

Heap Sort: 항상 O(n log n), 추가 메모리 O(1). 캐시 미스로 실제론 Merge Sort보다 느릴 수 있음.

Quick Sort: pivot-based divide and conquer. Average O(n log n), worst O(n²). Cache-friendly so usually fastest in practice. Swift's sort() is IntroSort (Quick + Heap + Insertion hybrid).

Merge Sort: always O(n log n), stable sort. Requires O(n) extra memory. Ideal for sorting linked lists.

Heap Sort: always O(n log n), O(1) extra space. Cache misses make it slower than Merge Sort in practice.

Swift CODE
// Quick Sort
func quickSort(_ arr: inout [Int], _ lo: Int, _ hi: Int) {
    guard lo < hi else { return }
    let p = partition(&arr, lo, hi)
    quickSort(&arr, lo, p-1); quickSort(&arr, p+1, hi)
}
func partition(_ arr: inout [Int], _ lo: Int, _ hi: Int) -> Int {
    let pivot = arr[hi]; var i = lo
    for j in lo..<hi { if arr[j] <= pivot { arr.swapAt(i, j); i += 1 } }
    arr.swapAt(i, hi); return i
}

// Merge Sort
func mergeSort(_ arr: [Int]) -> [Int] {
    guard arr.count > 1 else { return arr }
    let mid = arr.count / 2
    let left = mergeSort(Array(arr[..<mid]))
    let right = mergeSort(Array(arr[mid...]))
    return merge(left, right)
}
func merge(_ l: [Int], _ r: [Int]) -> [Int] {
    var res: [Int] = [], i = 0, j = 0
    while i < l.count && j < r.count {
        if l[i] <= r[j] { res.append(l[i]); i += 1 } else { res.append(r[j]); j += 1 }
    }
    return res + l[i...] + r[j...]
}

Binary Search
이분 탐색 · Lower/Upper Bound · Parametric Search
Binary search · Lower/Upper Bound · Parametric Search
개념CONCEPT

정렬된 배열에서 중간값과 비교하며 탐색 범위를 절반씩 줄임. 핵심은 left/right/mid 인덱스 관리와 종료 조건.

Lower Bound: target 이상의 첫 번째 인덱스. Upper Bound: target 초과의 첫 번째 인덱스. Parametric Search: "조건을 만족하는 최솟값" 문제.

Halve the search range each step by comparing with the middle element of a sorted array. The key is managing lo/hi/mid indices and termination conditions correctly.

Lower Bound: first index ≥ target. Upper Bound: first index > target. Parametric Search: find min/max value satisfying a monotone condition.

Swift CODE
// 기본 Binary Search
func binarySearch(_ arr: [Int], _ target: Int) -> Int? {
    var lo = 0, hi = arr.count - 1
    while lo <= hi {
        let mid = lo + (hi - lo) / 2  // 오버플로우 방지
        if      arr[mid] == target { return mid }
        else if arr[mid] <  target { lo = mid + 1 }
        else                       { hi = mid - 1 }
    }
    return nil
}

// Lower Bound (target 이상 첫 인덱스)
func lowerBound(_ arr: [Int], _ target: Int) -> Int {
    var lo = 0, hi = arr.count
    while lo < hi {
        let mid = lo + (hi - lo) / 2
        if arr[mid] < target { lo = mid + 1 } else { hi = mid }
    }
    return lo
}

// Parametric Search — "최소 몇 일?" 유형
func solve() -> Int {
    var lo = 1, hi = 1_000_000
    while lo < hi {
        let mid = lo + (hi - lo) / 2
        if canFinish(days: mid) { hi = mid } else { lo = mid + 1 }
    }
    return lo
}

Algorithm 03
Dynamic Programming
중복 부분 문제 + 최적 부분 구조 — 지수를 다항으로
Overlapping subproblems + optimal substructure — exponential to polynomial
개념CONCEPT

DP 적용 조건: ① 최적 부분 구조(큰 문제 최적해 = 작은 문제 최적해 조합) ② 중복 부분 문제(같은 하위 문제 반복).

Top-down (Memoization): 재귀 + 캐시 딕셔너리. Bottom-up (Tabulation): 작은 문제부터 테이블 채우기 — 공간 최적화 쉬움.

DP conditions: ① Optimal substructure (optimal solution for big problem = combination of optimal subproblem solutions). ② Overlapping subproblems (same subproblems solved repeatedly).

Top-down (Memoization): recursion + cache dict. Bottom-up (Tabulation): fill table from smallest subproblem — easier to optimize space.

// LCS (Longest Common Subsequence)
func lcs(_ a: [Character], _ b: [Character]) -> Int {
    let m = a.count, n = b.count
    var dp = [[Int]](repeating: [Int](repeating: 0, count: n+1), count: m+1)
    for i in 1...m { for j in 1...n {
        dp[i][j] = a[i-1] == b[j-1] ? dp[i-1][j-1]+1 : max(dp[i-1][j], dp[i][j-1])
    }}
    return dp[m][n]
}

// LIS O(n log n) — 이분 탐색 활용
func lis(_ nums: [Int]) -> Int {
    var tails: [Int] = []
    for n in nums {
        if let i = tails.firstIndex(where: { $0 >= n }) { tails[i] = n }
        else { tails.append(n) }
    }
    return tails.count
}

// 0-1 Knapsack
func knapsack(_ W: Int, _ wt: [Int], _ val: [Int]) -> Int {
    var dp = [Int](repeating: 0, count: W+1)
    for i in 0..<wt.count {
        for w in stride(from: W, through: wt[i], by: -1) {
            dp[w] = max(dp[w], dp[w-wt[i]] + val[i])
        }
    }
    return dp[W]
}

Algorithm 04
Greedy
지금 최선이 전체 최선 — 증명이 핵심
Local best = global best — proof is everything
개념CONCEPT

매 단계에서 지역적으로 최선인 선택. 전역 최적 보장 조건: Greedy Choice Property + 최적 부분 구조.

반례 주의: 동전 [1,5,6,9]원으로 11원을 거슬러야 할 때 Greedy(9+1+1=3개)는 최적이 아닙니다(5+6=2개). 증명 없이 사용하면 위험합니다.

Make the locally optimal choice at each step. For global optimality, Greedy Choice Property and Optimal Substructure must hold — always prove it first.

When Greedy fails: Coins [1,5,6,9] to make 11: Greedy picks 9+1+1 = 3 coins, but optimal is 5+6 = 2 coins. Greedy only works when large denominations are multiples of smaller ones.

// Activity Selection (종료 시간 순 정렬)
func activitySelection(_ intervals: [(start: Int, end: Int)]) -> Int {
    let sorted = intervals.sorted { $0.end < $1.end }
    var count = 0, lastEnd = Int.min
    for act in sorted {
        if act.start >= lastEnd { count += 1; lastEnd = act.end }
    }
    return count
}

// Fractional Knapsack
func fractionalKnapsack(_ W: Double, items: [(w: Double, v: Double)]) -> Double {
    let sorted = items.sorted { $0.v/$0.w > $1.v/$1.w }  // 단위가치 내림차순
    var remain = W, total = 0.0
    for item in sorted {
        if remain <= 0 { break }
        let take = min(remain, item.w)
        total += take * (item.v / item.w); remain -= take
    }
    return total
}

Algorithm 05
Backtracking
가지치기로 완전 탐색 최적화 — N-Queens · 순열 · 스도쿠
Optimizing brute-force via pruning — N-Queens · permutations · Sudoku
개념CONCEPT

가능한 모든 경우를 탐색하되 해가 될 수 없는 경우를 조기에 포기(Pruning)하는 전략. DFS 기반.

패턴: 선택 → 제약 확인 → 재귀 → 선택 취소(Undo). 상태를 선택 전으로 되돌리는 것이 핵심입니다.

Explore all possibilities but abandon ("backtrack") paths that cannot lead to a solution early — called Pruning. Based on DFS.

Pattern: Choose → Constrain → Recurse → Unchoose (Undo). Restoring state before the choice is the key.

// N-Queens
func solveNQueens(_ n: Int) -> Int {
    var cols = Set<Int>(), diag1 = Set<Int>(), diag2 = Set<Int>(), count = 0
    func bt(_ row: Int) {
        if row == n { count += 1; return }
        for col in 0..<n {
            if cols.contains(col) || diag1.contains(row-col) || diag2.contains(row+col) { continue }
            cols.insert(col); diag1.insert(row-col); diag2.insert(row+col)
            bt(row + 1)
            cols.remove(col); diag1.remove(row-col); diag2.remove(row+col)
        }
    }
    bt(0); return count
}

// 조합 (중복 없음, 순서 없음)
func combine(_ n: Int, _ k: Int) -> [[Int]] {
    var result: [[Int]] = [], cur: [Int] = []
    func bt(_ start: Int) {
        if cur.count == k { result.append(cur); return }
        for i in start...n { cur.append(i); bt(i+1); cur.removeLast() }
    }
    bt(1); return result
}

Algorithm 06
최단 경로
Dijkstra · Bellman-Ford · Floyd-Warshall
Dijkstra · Bellman-Ford · Floyd-Warshall
개념CONCEPT

Dijkstra: 음수 가중치 없는 그래프, 단일 출발점. Min Heap 사용 O((V+E) log V). 내비게이션, 라우팅 프로토콜에 사용.

Bellman-Ford: 음수 가중치 허용, 음수 사이클 감지. O(VE). V-1번 모든 간선 Relax.

Floyd-Warshall: 모든 쌍(All-pairs) 최단 경로. O(V³). 소규모 밀집 그래프 적합.

Dijkstra: No negative weights, single source. Min Heap, O((V+E) log V). Used in navigation and routing protocols.

Bellman-Ford: Allows negative weights, detects negative cycles. O(VE). Relax all edges V-1 times.

Floyd-Warshall: All-pairs shortest path. O(V³). Works with negative weights. Good for small dense graphs.

// Dijkstra (Min Heap)
func dijkstra(_ graph: [[(to: Int, w: Int)]], start: Int) -> [Int] {
    var dist = [Int](repeating: Int.max, count: graph.count)
    dist[start] = 0
    // (거리, 노드) 쌍을 Min Heap으로 관리
    var heap = [(dist: Int, node: Int)](); heap.append((0, start))
    while !heap.isEmpty {
        heap.sort { $0.dist < $1.dist }  // 실제론 MinHeap 사용
        let (d, u) = heap.removeFirst()
        if d > dist[u] { continue }
        for (v, w) in graph[u] {
            if dist[u] + w < dist[v] { dist[v] = dist[u] + w; heap.append((dist[v], v)) }
        }
    }
    return dist
}

// Floyd-Warshall
func floydWarshall(_ n: Int, edges: [(u: Int, v: Int, w: Int)]) -> [[Int]] {
    var dp = [[Int]](repeating: [Int](repeating: Int.max/2, count: n), count: n)
    for i in 0..<n { dp[i][i] = 0 }
    for e in edges { dp[e.u][e.v] = e.w }
    for k in 0..<n { for i in 0..<n { for j in 0..<n {
        dp[i][j] = min(dp[i][j], dp[i][k] + dp[k][j])
    }}}
    return dp
}
